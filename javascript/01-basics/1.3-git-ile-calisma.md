## 1.3 Git ile Çalışma

**Git**, kodun geçmişini tutan ve değişiklikleri kontrol etmeni sağlayan bir **versiyon kontrol sistemidir**.

Tek başına çalışırken de, ekipteyken de zorunludur.

---

### Git ne işe yarar?

- Kod geçmişini saklar
- Hatalı değişiklikten geri dönmeni sağlar
- Aynı projede birden fazla kişinin çalışmasını mümkün kılar

> Git = “kodun zaman makinesi”
> 

---

### Temel kavramlar

**Repository (repo)**

- Projenin Git ile takip edilen hali (Repository: Projedeki tüm dosyaların, yapılan değişikliklerin ve geçmişin Git tarafından takip edildiği ana klasör)

**Commit**

- Kodun anlık fotoğrafı
- “Ne yaptım?” sorusunun cevabı (Kodda o ana kadar yapılan değişikliklerin kaydedilmiş hâli; neyin eklendiğini, silindiğini veya düzeltildiğini gösteren kayıt)

**Branch**

- Ana koddan ayrılan çalışma alanı
- Deneme yaparken ana kodu bozmaz

---

## Temel Git akışı

1. Repo oluştur
2. Dosyaları değiştir
3. Değişiklikleri ekle
4. Commit at
5. Gerekirse branch aç
6. Merge et

Bu akış her projede aynıdır.

---

### En sık kullanılan komutlar

- `git init` → projeyi Git’e bağlar
- `git status` → durum kontrolü
- `git add .` → değişiklikleri sahneye alır
- `git commit -m "mesaj"` → kaydeder
- `git log` → geçmişi gör
- `git branch` → branch listesi
- `git checkout -b feature-x` → yeni branch

---

### Commit mesajı mantığı

İyi commit:

- kısa
- ne yaptığını anlatır

Örnek:

- `add login validation`
- `fix appointment time bug`

Kötü commit:

- `update`
- `fix`
- `asd`

---

### Branch mantığı (çok önemli)

- `main` / `master` → canlı, stabil kod
- `feature/*` → yeni özellik
- `bugfix/*` → hata düzeltme

> Main branch’e direkt geliştirme yapılmaz.
> 

---

### GitHub ile çalışma

GitHub = Git’in bulut tarafı

- Kod yedeklenir
- Projeler paylaşılır
- Pull Request ile kod incelenir

Temel akış:

- local → commit
- `git push`
- GitHub → PR → merge

---

### Neden erken öğrenilmeli?

- Büyük projelerde kaos önler
- Hata korkusunu azaltır
- Profesyonel yazılım kültürüdür

---

### Kısa özet

- Git = versiyon kontrol
- Commit = kod fotoğrafı
- Branch = güvenli çalışma alanı
- GitHub = paylaşım ve iş birliği
