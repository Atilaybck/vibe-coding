## 2.2 Veri Tipleri

JavaScript’te veri tipleri **iki ana gruba** ayrılır: **primitive** ve **reference**.

---

### Primitive (ilkel) tipler

Primitive tipler:

- **tek bir değer** tutar
- **değiştirilemezdir** (immutable)
- kopyalanırken **değer kopyalanır**

**Primitive türler**

- **string** → metin
    
    `"Merhaba"`
    
- **number** → sayı
    
    `42`, `3.14`
    
- **boolean** → doğru / yanlış
    
    `true`, `false`
    
- **null** → bilinçli olarak “boş” değer
    
    `null`
    
- **undefined** → henüz değer atanmamış
    
    `let x;`
    
- **bigint** → çok büyük sayılar
    
    `12345678901234567890n`
    
- **symbol** → benzersiz ve eşsiz değer
    
    `Symbol("id")`
    

---

### Reference (referans) tipler

Reference tipler:

- birden fazla değer tutabilir
- bellekte **adres (referans)** ile saklanır.
    
    (Verinin kendisi değil, RAM’de durduğu yerin adresi tutulur — **örn:** arr1 = [1,2,3] RAM’de 0xA12 adrestedir, arr2 = arr1 yapılınca arr2 de 0xA12 adresini gösterir; arr2.push(4) → ikisi de [1,2,3,4] olur)
    
- kopyalanınca **adres kopyalanır**

**Reference türler**

- **object** → `{}`
- **array** → `[]`
- **function** → `function() {}`

```jsx
const user = {name:"Atılay" };
```

---

### Primitive vs Reference farkı (çok önemli)

```jsx
let a =5;
let b = a;
b =10;
// a hâlâ 5
```

```jsx
const obj1 = {age:25 };
const obj2 = obj1;
obj2.age =30;
// obj1.age = 30
```

---

### `typeof` nedir?

`typeof`, bir değerin **hangi veri tipinde olduğunu** söyler.

```jsx
typeof"hello";// "string"
typeof10;// "number"
typeoftrue;// "boolean"
```

---

### `typeof` tuzakları

### En meşhur tuzak: `typeof null`

```jsx
typeofnull;// "object" ❌

```

**Neden?**

Bu, JavaScript’in **tarihi bir hatasıdır**.

`null` aslında **primitive** bir tiptir ama `typeof` onu yanlışlıkla `"object"` olarak döndürür.

> Bu hata geriye uyumluluk için düzeltilmemiştir.
> 

---

### Diğer küçük tuzaklar

```jsx
typeof [];// "object"
typeof {};// "object"
typeof function(){};// "function"
```

> function, teknik olarak object’tir ama typeof onu özel ayırır.
> 

---

### Kısa özet

- Primitive → tek değer, değer kopyalanır
- Reference → adres kopyalanır
- `typeof` tipi söyler ama **her zaman güvenilir değildir**
- `typeof null === "object"` → ezberle
