## 4.1 Fonksiyon Temelleri

**Fonksiyon**, tekrar kullanılabilir bir kod bloğudur.

“Bir işi tanımla, gerektiğinde çağır” mantığıyla çalışır.

---

### Function Declaration (Fonksiyon Bildirimi)

Fonksiyon **önceden tanımlıdır**, kodun her yerinden çağrılabilir.

```jsx
function greet(name) {
  return `Merhaba ${name}`;
}
```

- Hoisting vardır (tanımlanmadan önce çağrılabilir yani, JavaScript, fonksiyon tanımını kod çalışmadan önce belleğe alır; bu yüzden fonksiyon, kodda daha aşağıda tanımlı olsa bile üst satırlardan çağrılabilir)
- Okunaklı ve klasik yöntem

---

### Function Expression (Fonksiyon ifadesi)

Fonksiyon bir değişkene atanır.

```jsx
const greet = function (name) {
  return `Merhaba ${name}`;
};

```

- Hoisting yok
- Daha kontrollü kullanım
- Modern projelerde sık görülür

---

### Declaration vs Expression farkı

- Declaration → erken çağrılabilir
- Expression → tanımlandıktan sonra
- Mantık aynı, davranış farklı

---

### Parametre nedir?

Parametre, fonksiyonun **dışarıdan aldığı veri**dir.

```jsx
function sum(a, b) {
  return a + b;
}
```

- `a` ve `b` → parametre

---

### Return nedir?

`return`, fonksiyonun **ürettiği sonucu dışarı vermesidir**.

```jsx
function double(x) {
  return x * 2;
}
```

- `return` sonrası fonksiyon biter
- `return` yoksa → `undefined`

---

### Default Params (varsayılan parametre)

Parametre gönderilmezse **otomatik değer alır**.

```jsx
function greet(name = "Misafir") {
  return `Merhaba ${name}`;
}
```

```jsx
greet();       // "Merhaba Misafir"
greet("Ali");  // "Merhaba Ali"

```

---

### Küçük ama önemli notlar

- Fonksiyon tek iş yapmalı
- Parametre sayısı az olmalı
- `return` net olmalı

---

### Kısa özet

- Function declaration → klasik
- Function expression → değişkene atanır
- Parametre → giriş
- Return → çıkış
- Default param → yedek değer
